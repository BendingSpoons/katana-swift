  • replaceKey Default implementation

    The replace key of the description. When two descriptions have the same replaceKey, Katana consider them interchangeable. During an UI update, Katana will try to minimise the creation of new nodes (and therefore UIView/NSView instances) by reusing old nodes. This is only possible when the old node description is interchangeable with the node that Katana is trying to render.

    You can customise the replace key to control the reuse of nodes and views.

    Default Implementation

    The default implementation of this method returns a value that allow Katana to exchange nodes related to the same description types. If the description has Keyable properties, the key is used as an extra source of information.
